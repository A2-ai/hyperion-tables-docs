## Description

Summary specification for run summary tables

## Usage

```r
SummarySpec(
  title = "Run Summary",
  models_to_include = NULL,
  tag_filter = NULL,
  summary_filter = summary_filter_rules(),
  remove_unrun_models = TRUE,
  columns = NULL,
  add_columns = NULL,
  drop_columns = NULL,
  hide_empty_columns = TRUE,
  n_sigfig = 3,
  n_decimals_ofv = 3,
  time_format = "seconds",
  pvalue_scientific = FALSE,
  pvalue_threshold = NULL,
  footnote_order = "abbreviations"
)
```

<tr>
<td><code>title</code></td>

<td>Character vector of model names to include in the table (with or without .mod/.ctl extensions), or NULL (default).</td>

<td>Filter rules created with <code>summary_filter_rules()</code>.</td>

<tr>
<td><code>remove_unrun_models</code></td>
<td>Logical. If TRUE (default), models without completed runs are excluded from the table.</td>

<td>Character vector of columns to include. Valid columns: "based_on", "description", "n_parameters", "problem", "number_data_records", "number_subjects", "number_obs", "estimation_method", "estimation_time", "covariance_time", "postprocess_time", "function_evaluations", "significant_digits", "ofv", "dofv", "condition_number", "termination_status", "pvalue", "df". Note: "pvalue" and "df" require "dofv" to be calculated; pvalue uses the Likelihood Ratio Test (LRT) assuming nested models.</td>

<tr>
<td><code>add_columns</code></td>
<td>Character vector of columns to append to the default <code>columns</code> list, or NULL (default).</td>

<tr>
<td><code>hide_empty_columns</code></td>

<tr>

<tr>

<td>Format for time columns. Options: "seconds" (default), "minutes", "hours", "auto" (auto-scale based on magnitude).</td>
</tr>
