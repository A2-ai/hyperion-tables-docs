## Description

Controls which models appear in the summary table based on filter
expressions evaluated against summary columns.

## Details

The built-in `SummarySpec` method interprets unnamed arguments in `...` as
summary filter rule expressions. Named arguments in `...` are ignored with a
warning.






## Usage

```r
set_spec_summary_filter(spec, ..., overwrite = FALSE)
```

## Arguments

<div class="arg-table">
<table>
<thead>
<tr><th>Argument</th><th>Description</th></tr>

<td><code>spec</code></td>

</tr>

<td>Additional arguments passed to methods.</td>
</tr>
<tr>
<td><code>overwrite</code></td>
<td>If FALSE (default), append to existing rules. If TRUE, replace all existing rules.</td>
</tr>

</div>


## Value

Modified spec




## Examples

```r
spec <- SummarySpec() |>
  set_spec_summary_filter(ofv < 1000)
```
