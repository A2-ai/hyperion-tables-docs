## Description

Creates rules for assigning parameters to named sections in the output table. Rules are evaluated after name transformation, so you can match on the final display name or use the preserved `nonmem_name` and `user_name` columns.

## Usage

```r
section_rules(...)
```

## Value

List of quosures for use in TableSpec

## Available columns

The following columns are available for use in section rules: 
- `nonmem_name` - NONMEM identifier ("THETA1", "OMEGA(1,1)")
- `user_name` - User name from control file comments ("CL", "OM1")
- `name` - Display name (depends on `parameter_names` setting)
- `kind` - Parameter type: "THETA", "OMEGA", or "SIGMA"
- `diagonal` - TRUE for diagonal matrix elements (variance), FALSE for off-diagonal (covariance)
- `fixed` - TRUE if parameter is fixed

## Examples

```r
section_rules(
  grepl("~", user_name) ~ "Covariate Effects",
  kind == "THETA" ~ "Structural Parameters",
  kind == "OMEGA" & diagonal ~ "Between-Subject Variability",
  kind == "SIGMA" ~ "Residual Variability"
)
```
